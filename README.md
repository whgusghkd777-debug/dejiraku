  Deziraku (デジラク) - ホテル予約システム
  安定したデータ処理と直感的な予約体験を提供する宿泊予約プラットフォーム Role: メインバックエンド開発 (Main Backend Developer)
  1. プロジェクト概要
  「Deziraku」は、ユーザーが宿泊施設を検索し、リアルタイムで予約および決済を行えるWebサービスです。
  バックエンドのメイン開発者として、データベース設計から予約・決済ロジックの構築まで、システムの中核となる部分を主導しました。

  3. 技術スタック
  Language/Framework: Java 1.8 / Spring Framework
  Server: Apache Tomcat 9
  Database: Oracle DB
  Frontend: HTML5, CSS3, JavaScript, JSP, Bootstrap 5
  Design Pattern: MVC Pattern

  4. 担当した主な機能 (Key Contributions)
     バックエンドの設計から実装まで、以下の業務を遂行しました。
     予約管理システムの構築 : チェックイン・アウトの日付および人数に基づいたリアルタイム在庫確認ロジックを実装。
     予約の重複を防ぐためのデータ整合性検証ロジックを設計。

  決済プロセスとトランザクション管理:  予約情報の登録と決済処理をひとつのトランザクションとして管理し、データの不一致を防止。
  データベース設計 (DB Design):  拡張性を考慮したテーブル構造の設計およびERDの作成。ホテル、客室、予約、ユーザー間のリレーションシップを最適化。

  5. 技術的な課題と解決策 (Technical Challenges)
  データ整合性の確保 (Concurrency Control)
  課題: 複数のユーザーが同時に同一の客室を予約しようとした際、データの競合が発生するリスクがありました。

  解決策: 楽観的ロック (Optimistic Lock) の概念を取り入れ、決済確定直前に再度在庫状態を確認するバリデーションを導入。
        これにより、二重予約を未然に防ぎ、システムの信頼性を向上させました。

  検索クエリの最適化 (SQL Tuning)
  課題: 検索条件（地域、価格、日付など）が複雑になるほど、Oracle DBからの応答速度が低下する問題がありました。

  解決策: よく使用される検索カラムに対してインデックス (Index) を適用し、MyBatisのSQL文をチューニングすることで、照会速度の大幅な向上を実現しました。

  6. 自己評価と今後の展望
  今回のプロジェクトを通じ、単に機能を実装するだけでなく、**「いかに安全にデータを管理するか」**
  というバックエンドエンジニアとしての責務を深く学ぶことができました。
  今後は、よりモダンな技術スタック（Spring Boot, JPA）へのリファクタリングや、
  AWS等のクラウド環境へのデプロイにも挑戦し、インフラまで理解できるエンジニアを目指します。
